| `Attack` | `settings/dist/Attack/` | 攻击/弹幕参数 |
| `Skill` | `settings/dist/Skill/` | 记忆结晶 |
| `SpellCard` | `settings/dist/SpellCard/` | 符卡 |

| `Stage1` | `settings/dist/StageProcess/` | 第一关刷怪事件 |
| `Upgrade` | `settings/dist/Upgrade/` | 升级项定义 |

| `danma` / `d4c` | `settings/dist/Danmaku*` | 弹幕 |

局内读取被升级修改过的行：

```gdscript
# 推荐
var row = RunModifiers.resolve("Attack", attack_id)
# 或 table 上的封装，例如 resolve_attack
```

基表只读：

```gdscript
var base = table.get_base_row("Skill", skill_id)
```

## SignalBus（常用信号入口）

在 `gamemanager/SignalBus/SignalBus.gd` 中定义。改玩法时建议 **全局搜索** 信号名：

- 技能：`try_add_skill`、`skill_upgrade`…
- 符卡：`try_add_card`、`card_upgrade`…
- 被动：`try_add_passive`…
- 刷怪：`add_mob_to_manager`
- Boss：`boss_stage`、`kill_all`

## GUI 视图 id

视图在 `ui/game.tscn` 的 `GUIViewManager.viewConfigList` 中配置。代码里 `open_view("InGame")` 的字符串需与 `GUIViewConfig.id` 一致。常见 id：

`StartMenu`、`StoryModeMenu`、`InGame`、`Hud`、`LevelUp`、`PauseMenu`、`ClearMenu`、`TestScene`、`SettingsMenu`、`ShopMenu`、`WikiMenu`、`DialogueMenu`

## 铁律（避免常见 bug）

1. 局内 **不要** 写 `table.Attack[id].field = x` → 用 `RunModifiers.set_row_fields`（见 `upgrade.gd`）。
2. 新局 **不要** 只清 UI → 必须 `table.clear_session_state()` + `RunSession.end_run()`。
3. 新代码 **优先** `RunSession`，少用 `player_var.SkillManager` 转发。
